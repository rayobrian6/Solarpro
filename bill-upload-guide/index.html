Step-by-Step Data Flow
What happens from upload click to structured output
1
User Uploads File (or enters manually)
The BillUploadFlow component accepts drag-and-drop or file picker input. Supported formats: PDF, JPG, JPEG, PNG, JFIF, WEBP. A manual entry fallback accepts raw kWh + address text. The file is sent as multipart/form-data to POST /api/bill-upload.
BillUploadFlow.tsx FormData multipart/form-data
2
Auth Check + File Type Routing
The API route authenticates the user via getUserFromRequest(). It then reads the file's MIME type and extension to route to the correct extractor: extractPdfText() for PDFs or extractImageText() for images. Unsupported types return a 400 error immediately.
api/bill-upload/route.ts JWT Auth MIME routing
3
Text Extraction (PDF or Image)
PDF path: pdf-parse v2 is tried first (class-based API), then v1 (default function), then pdftotext CLI as a local fallback.

Image path: Google Cloud Vision API (DOCUMENT_TEXT_DETECTION) is tried first using your GOOGLE_MAPS_API_KEY. If Vision fails or returns insufficient text, tesseract CLI is tried as a local fallback.

If no text is extracted after all attempts, a 422 error is returned with a helpful hint.
pdf-parse v2 Google Vision API Tesseract (local) pdftotext (local)
4
Regex-Based Bill Parsing
parseBillText() in lib/billOcr.ts runs a battery of regex patterns against the extracted text to find: utility provider (37+ known utilities), kWh usage (monthly + annual), electricity rate ($/kWh), total amount due, billing period dates, customer name, service address, and account number. A confidence score (high/medium/low) is computed based on how many fields were found.
lib/billOcr.ts 37+ utility patterns 8 field extractors Confidence scoring
5
Geocoding + Utility Detection
If a service address was extracted, geocodeAddress() resolves it to lat/lng + full location metadata using the Census Bureau Geocoding API (free, no key) with Google Maps as fallback. The coordinates are then passed to detectUtility() which queries the NREL URDB API for the exact utility serving that location, with a comprehensive 50-state fallback dataset.
lib/locationEngine.ts lib/utilityDetector.ts Census Bureau API NREL URDB
6
System Sizing + Response
Using the extracted annual kWh and the geocoded latitude, calculateRecommendedSystemSize() computes the recommended solar system size in kW using location-adjusted peak sun hours (3.6–5.5 hrs/day) and 80% system efficiency. The full structured response is returned: billData, validation, locationData, utilityData, and systemSizing.

// POST /api/bill-upload — accepts multipart/form-data
export async function POST(req: NextRequest) {
  // 1. Auth guard
  const user = await getUserFromRequest(req);
  if (!user) return NextResponse.json({ success: false, error: 'Unauthorized' }, { status: 401 });

  // 2. Parse form data — accepts 'file' (binary) or 'text' (manual entry)
  const formData = await req.formData();
  const file = formData.get('file') as File | null;
  const rawText = formData.get('text') as string | null;

  // 3. Route to correct extractor based on MIME type
  if (fileType === 'application/pdf') {
    extractedText = await extractPdfText(buffer);     // pdf-parse → pdftotext
  } else if (fileType.startsWith('image/')) {
    extractedText = await extractImageText(buffer, fileType); // Vision → Tesseract
  }

  // 4. Parse + validate
  const billData = parseBillText(extractedText);
  const validation = validateBillData(billData);

  // 5. Geocode + detect utility (if address found)
  if (billData.serviceAddress) {
    const geoResult = await geocodeAddress(billData.serviceAddress);
    const utilityResult = await detectUtility(lat, lng, stateCode, city);
  }

  // 6. Calculate system size + return
  return NextResponse.json({ success: true, billData, validation, locationData, utilityData, systemSizing });
}

PDF Text Extraction
Handling text-based and scanned PDFs on Vercel
📄 Text-Based PDFs

Most utility bills downloaded from utility portals are text-based PDFs. pdf-parse extracts the embedded text directly — fast, accurate, no OCR needed. This is the primary path and works perfectly on Vercel.

🖼️ Scanned PDFs

Scanned PDFs contain images of text, not actual text. pdf-parse returns empty or near-empty text. The current code falls back to pdftotext CLI (local only). Production fix: convert PDF pages to images and run through Google Vision.

📄 app/api/bill-upload/route.ts — Current PDF extractor (already in your code) TS 
async function extractPdfText(buffer: Buffer): Promise<string> {
  try {
    // Dynamic import avoids Next.js bundling issues
    const pdfParseModule = await import('pdf-parse');

    // pdf-parse v2: exports PDFParse class (breaking change from v1)
    const PDFParseClass = (pdfParseModule as any).PDFParse;
    if (PDFParseClass) {
      const parser = new PDFParseClass();
      const data = await parser.parse(buffer);
      return data.text || '';
    }

    // pdf-parse v1 fallback: default export is a function
    const pdfParseFn = (pdfParseModule as any).default || pdfParseModule;
    if (typeof pdfParseFn === 'function') {
      const data = await pdfParseFn(buffer, { max: 0 });
      return data.text || '';
    }
    throw new Error('pdf-parse: no usable export found');
  } catch (err: any) {
    console.error('[pdf-parse]', err.message);
    return extractPdfTextCli(buffer); // local fallback only
  }
}

Scanned PDF Gap — Production Improvement

When pdf-parse returns less than ~50 characters from a PDF, it's likely a scanned document. Add this detection and route scanned PDFs through Google Vision for much better results.

📄 Improved extractPdfText() — handles scanned PDFs via Vision API TS 
async function extractPdfText(buffer: Buffer): Promise<string> {
  // Step 1: Try pdf-parse for text-based PDFs
  let text = '';
  try {
    const pdfParseModule = await import('pdf-parse');
    const PDFParseClass = (pdfParseModule as any).PDFParse;
    if (PDFParseClass) {
      const data = await (new PDFParseClass()).parse(buffer);
      text = data.text || '';
    } else {
      const fn = (pdfParseModule as any).default || pdfParseModule;
      if (typeof fn === 'function') text = ((await fn(buffer)).text) || '';
    }
  } catch { /* fall through */ }

  // Step 2: If text is sparse (<50 chars), it's likely a scanned PDF
  // Route through Google Vision for OCR on the first page image
  if (text.trim().length < 50) {
    console.log('[bill-upload] Sparse PDF text — attempting Vision OCR');
    // Convert first PDF page to image using pdf2pic (optional dependency)
    try {
      const { fromBuffer } = await import('pdf2pic');
      const converter = fromBuffer(buffer, { density: 200, format: 'png', width: 1700, height: 2200 });
      const page = await converter(1, { responseType: 'buffer' });
      if (page.buffer) {
        text = await extractImageText(page.buffer as Buffer, 'image/png');
      }
    } catch {
      // pdf2pic not installed — fall back to CLI
      text = await extractPdfTextCli(buffer);
    }
  }

  return text;
}

📄 app/api/bill-upload/route.ts — Image OCR (already in your code) TS 
async function extractImageText(buffer: Buffer, mimeType: string): Promise<string> {
  // Primary: Google Cloud Vision API — DOCUMENT_TEXT_DETECTION
  // Best for dense text documents like utility bills
  const googleKey = process.env.GOOGLE_MAPS_API_KEY;
  if (googleKey) {
    const base64 = buffer.toString('base64');
    const res = await fetch(
      `https://vision.googleapis.com/v1/images:annotate?key=${googleKey}`,
      {
        method: 'POST',
        body: JSON.stringify({
          requests: [{
            image: { content: base64 },
            features: [{ type: 'DOCUMENT_TEXT_DETECTION', maxResults: 1 }],
          }],
        }),
        signal: AbortSignal.timeout(20000),
      }
    );
    const data = await res.json();
    const text = data?.responses?.[0]?.fullTextAnnotation?.text;
    if (text?.trim().length > 20) return text; // ✓ success
  }

  // Fallback: Tesseract CLI (local/sandbox only — not available on Vercel)
  // ... tesseract execSync call
  return '';
}

// Add to lib/billOcr.ts — extract demand charge for commercial bills
function extractDemandCharge(text: string): number | undefined {
  const patterns = [
    /demand\s+charge[:\s]+\$([0-9,]+\.[0-9]{2})/i,
    /(?:peak\s+)?demand[:\s]+([0-9.]+)\s*kw/i,
    /\$([0-9,]+\.[0-9]{2})\s+demand/i,
  ];
  for (const pattern of patterns) {
    const match = text.match(pattern);
    if (match) {
      const val = parseFloat(match[1].replace(/,/g, ''));
      if (val > 0 && val < 10000) return val;
    }
  }
  return undefined;
}

Recommended Improvements
Prioritized enhancements to increase accuracy and reliability
🤖
LLM Fallback — GPT-4o or Claude
When regex extraction yields low confidence (fewer than 2 fields), pass the raw text to GPT-4o with a structured prompt asking it to extract kWh, rate, address, and utility as JSON. This handles unusual bill formats, international utilities, and edge cases that regex can't cover. Estimated improvement: +30–40% accuracy on low-confidence bills.
High Priority
📄 lib/billOcr.ts — Add LLM fallback (pseudocode) TS 
import OpenAI from 'openai';
const openai = new OpenAI();

export async function parseBillTextWithLLM(text: string): Promise<BillExtractResult> {
  // First try fast regex parsing
  const regexResult = parseBillText(text);

  // If confidence is low, use LLM as fallback
  if (regexResult.confidence === 'low' && process.env.OPENAI_API_KEY) {
    try {
      const completion = await openai.chat.completions.create({
        model: 'gpt-4o-mini',  // cheap + fast for structured extraction
        response_format: { type: 'json_object' },
        messages: [{
          role: 'system',
          content: `Extract utility bill data as JSON with these fields:
            monthlyKwh (number), annualKwh (number), electricityRate ($/kWh),
            totalAmount ($), utilityProvider (string), serviceAddress (string),
            customerName (string), accountNumber (string),
            billingPeriodStart (string), billingPeriodEnd (string).
            Return null for any field not found. Only return valid JSON.`
        }, {
          role: 'user',
          content: `Extract data from this utility bill:\n\n${text.substring(0, 3000)}`
        }],
      });

      const llmData = JSON.parse(completion.choices[0].message.content || '{}');

      // Merge LLM results with regex results (regex takes precedence)
      return {
        ...regexResult,
        monthlyKwh: regexResult.monthlyKwh || llmData.monthlyKwh,
        electricityRate: regexResult.electricityRate || llmData.electricityRate,
        utilityProvider: regexResult.utilityProvider || llmData.utilityProvider,
        serviceAddress: regexResult.serviceAddress || llmData.serviceAddress,
        confidence: 'medium',  // upgrade from 'low'
      };
    } catch (err) {
      console.warn('[billOcr] LLM fallback failed:', err);
    }
  }

  return regexResult;
}
